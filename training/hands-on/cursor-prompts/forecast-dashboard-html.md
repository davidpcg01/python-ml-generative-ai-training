# Prompt: Create Static HTML Forecast Dashboard

Use this prompt inside `Cursor` after the forecast CSV files already exist.

```text
You are acting as a petroleum-engineering analytics assistant inside a Cursor project.

I need you to create a static HTML dashboard that displays the production-forecast results in both numerical and graphical form.

Dataset note:
- `oil_bbl` is monthly volume.
- The notebook and forecast files compare `oil_rate_bbl_per_day`, which is a daily rate calculated from `days_on_prod`.

Available project inputs:
- results/production_forecasting/holdout_forecasts.csv
- results/production_forecasting/prediction_timeline.csv
- results/production_forecasting/arps_parameters.csv
- results/production_forecasting/model_comparison_metrics.csv
- results/production_forecasting/per_completion_metrics.csv
- optionally:
  - results/production_forecasting/per_well_metrics.csv
  - results/production_forecasting/rf_model_summary.csv
  - results/production_forecasting/time_series_model_summary.csv
  - results/production_forecasting/arps_holdout_forecast.csv
  - results/production_forecasting/arps_model_metrics.csv

Goal:
Build a GitHub-friendly static dashboard that can be opened locally in a browser without any backend server.

Required output:
- Create a file named forecast_dashboard.html

Dashboard requirements:
1. Use only HTML, CSS, and vanilla JavaScript.
2. Do not use React, Vue, or a build step.
3. You may use a CDN library if needed for charting, preferably Chart.js.
4. The dashboard must load data from local CSV files in the project or from embedded JSON generated by a companion script.
5. If browser restrictions make direct CSV loading unreliable, create a small Python helper script named build_dashboard_data.py that converts the CSV inputs into a single JSON file named dashboard_data.json.

Visual requirements:
- The page should feel professional and engineering-oriented
- Prefer a clean dark or muted industrial theme
- Make it suitable for internal technical review, not marketing
- Use clear axis labels, units, legends, and section headings

Required dashboard sections:
1. Executive summary cards
   - total completions analyzed
   - average MAE
   - average RMSE
   - best-performing completion and model
   - worst-performing completion and model
2. Per-completion forecast plot
   - actual vs forecasted oil rate over the 12-month forecast months
   - include `Arps`, `Random forest`, `Exponential smoothing`, and `ARIMA`
   - one chart per selected completion or a dropdown completion selector
3. Per-completion train/test/forecast comparison plot
   - actual vs predicted oil rate across the full selected 48-month window
   - visually distinguish train, test, and forecast periods
4. Error summary table
   - well_id
   - phase
   - MAE
   - RMSE
   - MAPE
   - fit status if available
5. Arps parameter table
   - qi
   - di
   - b
6. Data export section
   - links or clear references to the CSV files used

Numerical requirements:
- Show rates with reasonable rounding
- Handle missing values safely
- Do not break if one well has no valid fit

Engineering requirements:
- Label forecast lines clearly as forecast, not actual
- Label the internal test period clearly so it is not confused with the final 12-month forecast
- Make the per-completion view easy to compare
- Use units explicitly such as `bbl/day on production` for rates and `bbl/month` for monthly volumes where applicable
- Use the updated model names consistently: `Arps hyperbolic`, `Random forest (multivariate)`, `Exponential smoothing`, and `ARIMA`
- Include a note that the dashboard is based on holdout forecast comparison rather than a live real-time deployment

File-creation rules:
- Create only the files necessary for the dashboard
- If you create a helper script or JSON, name them clearly
- Do not overwrite unrelated project files
- Do not create placeholder charts

Validation requirements:
- Make sure the HTML opens directly in a browser
- If a local-file restriction exists for CSV loading, solve it with the helper JSON build step
- Include brief run instructions at the top of the HTML file as an HTML comment or in a small footer note

At the end:
- explain which files you created
- explain how to generate the dashboard data if a helper script is used
- explain how to open the dashboard locally
```
